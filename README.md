# CNN Overview – Understanding Convolution and Pooling from Scratch

This project documents my foundational exploration of convolutional neural networks (CNNs) using PyTorch. The goal was to understand how convolution, padding, stride, manual filters, and pooling operations affect image data and feature extraction.

All experiments were performed using sample RGB images loaded from sklearn.

---

## Project File

- cnn_overview.py – Hands-on exploration of convolution and pooling operations

---

# Objective

The goals of this work were:

- Understand tensor shape transformations for CNNs
- Explore convolution operations step by step
- Visualize feature maps generated by convolution
- Study the effect of padding and stride
- Manually implement convolution using torch.nn.functional
- Create custom filters
- Compare Max Pooling and Average Pooling

---

# Dataset Preparation

Sample images were loaded using:

- sklearn.datasets.load_sample_images

Steps performed:

1. Stacked images into a NumPy array
2. Converted to PyTorch tensor
3. Normalized pixel values to range [0, 1]
4. Permuted tensor dimensions from:
   
   (N, H, W, C) → (N, C, H, W)

This format is required for PyTorch Conv2d layers.

---

# Image Cropping

Applied:

- CenterCrop((70, 120))

Purpose:

- Standardize spatial dimensions
- Prepare consistent input size for convolution

---

# Basic Convolution

Created a convolution layer:

- in_channels = 3
- out_channels = 32
- kernel_size = 7
- No padding

Observed:

- Output feature maps were smaller than input
- Feature maps detected spatial patterns (edges and structures)

This demonstrated how convolution extracts spatial features.

---

# Convolution with Same Padding

Modified convolution:

- padding = "same"

Observed:

- Output size matches input size
- Border information preserved
- No spatial shrinkage

This demonstrated the importance of padding.

---

# Convolution with Stride

Modified convolution:

- stride = 2
- padding = 3

Observed:

- Spatial resolution reduced by half
- Information loss compared to stride=1
- Image quality in feature maps decreased

This showed how stride controls downsampling.

---

# Inspecting Parameters

Examined:

- conv_layer.weight.shape
- conv_layer.bias.shape

Learned:

- Each output channel has its own filter
- Filters are shared spatially across the image

---

# Manual Convolution with torch.nn.functional

Used:

- F.conv2d()

Manually created:

- Random filters
- Zero biases

Purpose:

- Understand how convolution works internally
- Control weights directly

---

# Custom Vertical and Horizontal Filters

Created manual filters:

- Vertical line detector
- Horizontal line detector

Observed:

- Vertical filter emphasized vertical structures
- Horizontal filter emphasized horizontal structures
- Zero padding introduced dark borders

This demonstrated how specific kernels detect specific spatial patterns.

---

# Pooling Operations

## Max Pooling

- kernel_size = 2

Observed:

- Preserved strong edges
- Highlighted dominant features
- Reduced spatial size

## Average Pooling

- kernel_size = 2

Observed:

- Smoothed the image
- Reduced sharp edges
- Averaged local pixel values

Comparison:

- Max pooling emphasizes important features
- Average pooling smooths features

---

# Key Learnings

- CNNs require channel-first tensor format
- Convolution extracts spatial features using shared kernels
- Padding preserves spatial dimensions
- Stride reduces spatial resolution
- Manual filters help understand kernel behavior
- Max pooling highlights dominant activations
- Average pooling smooths spatial features

---

# Conclusion

This project builds foundational intuition for convolutional neural networks by directly visualizing feature maps and manually controlling convolution operations. It establishes a strong understanding of how CNNs process images before moving into deeper architectures.
